# NostalgiaHub – Sistema de Gestión de Colecciones Nostálgicas

## Descripción del Sistema

**NostalgiaHub** es un sistema diseñado para permitir a los usuarios organizar y gestionar objetos coleccionables con valor sentimental o histórico, agrupándolos en diferentes categorías.

El sistema permite registrar, visualizar, editar y eliminar elementos pertenecientes a una colección, facilitando la administración de objetos nostálgicos de manera organizada.

Las colecciones pueden incluir diferentes tipos de objetos, por ejemplo:

* videojuegos retro
* monedas antiguas
* juguetes clásicos
* discos o vinilos
* revistas antiguas
* cartas coleccionables

La aplicación permite que los usuarios organicen sus objetos por categorías, registren información relevante de cada elemento y consulten fácilmente el contenido de sus colecciones.

---

## Problema

Muchas personas poseen colecciones de objetos nostálgicos con valor sentimental o histórico, pero no cuentan con una herramienta sencilla que les permita administrarlas de manera organizada.

Actualmente es común utilizar métodos informales como:

* listas en papel
* notas en dispositivos móviles
* archivos de texto

Estos métodos presentan dificultades como:

* falta de organización en la información
* dificultad para buscar elementos específicos
* problemas para actualizar los registros

Por esta razón, se propone el desarrollo de un sistema que permita gestionar colecciones nostálgicas de forma más organizada y accesible.

---

## Alcance del Sistema

El sistema permitirá realizar las siguientes acciones:

* Crear categorías de colección.
* Registrar elementos dentro de una categoría.
* Visualizar las categorías existentes.
* Visualizar los elementos registrados en una categoría.
* Editar la información de elementos o categorías.
* Eliminar elementos o categorías.
* Buscar elementos dentro de las colecciones.
* Mostrar la cantidad de elementos registrados por categoría.

El sistema estará enfocado en la gestión básica de colecciones, sin incluir funcionalidades avanzadas como autenticación de usuarios o almacenamiento en bases de datos externas.

---

## Actores del Sistema

### Usuario

El **usuario** es la persona que interactúa con el sistema para gestionar sus colecciones nostálgicas.

El usuario podrá:

* crear categorías de colección
* registrar elementos dentro de una categoría
* visualizar las colecciones registradas
* editar información de elementos o categorías
* eliminar registros existentes
* buscar elementos dentro de las colecciones
* consultar la cantidad de elementos por categoría

---

## Funcionalidades del Sistema

El sistema contará con las siguientes funcionalidades principales:

1. Crear categorías de colección.
2. Registrar elementos dentro de una categoría.
3. Visualizar categorías registradas.
4. Visualizar elementos de una categoría.
5. Editar información de categorías o elementos.
6. Eliminar elementos o categorías.
7. Buscar elementos dentro de las colecciones.
8. Mostrar la cantidad de elementos registrados por categoría.

---

## Reglas del Sistema

El sistema debe cumplir con las siguientes reglas:

* No se deben permitir categorías duplicadas.
* Los elementos solo pueden agregarse a categorías existentes.
* Para editar un elemento, el usuario debe seleccionarlo previamente.
* Los campos del elemento deben contener información válida antes de ser registrados.
* Al eliminar una categoría, también se eliminarán los elementos asociados a ella.
